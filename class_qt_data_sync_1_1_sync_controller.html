Detailed Description

A controller class to monitor and control a datasync instance.

With this controller, you can report the sychronization state to the user, and you can trigger specific actions.

◆ triggerResync

QtDataSync::SyncController::triggerResync ( )
slot

Tells the datasync instance to start a resynchronization.

Triggering resync means both the local and the remote change state (not the data!) are discarded and then set to changed for all existing entries, followed by a synchronization. A resync can be used to recover errors in remote or local state. After a resync you can be shure both sides have the same data.

Attention
Triggering a resync may or may not work. All it does is request a resync at the remote. Only if the remote accepts, resync will actually work. If the remote does not allow it, a normal sync will be performed
See also
QtDataSync::SyncController::triggerResyncWithResult, QtDataSync::SyncController::triggerSync

◆ triggerSync

QtDataSync::SyncController::triggerSync ( )
slot

Tells the datasync instance to start synchronizing.

Triggering sync means that the instance will simply reload the remotes state and compare it with the local state. Typically, this happens automatically on creation of the instance and is repeated while running. However, in some cases you may want to explicitly trigger a synchronization.

QtDataSync::SyncController::authenticationError
read

Holds authentication errors, if they did occur.

Default: empty (default constructed)

If this property is empty, it means there is no error. If authentication failed inside the remote connector, this property will contain the error string (and SyncController::syncState will be SyncController::Disconnected)

QtDataSync::SyncController::syncEnabled
readwrite

Default: true

If set to false, the engone will disconnect from any servers and stay this ways. It's basically an "offline mode". Please not that changes are still locally stored, and once sync gets enabled, all changes since the last connection will be synchronized.
